# Синхронизация документации с OpenAPI субмодулей

В Python-сервисах (`services/*`) спецификация **OpenAPI 3** отдаётся в рантайме FastAPI:

- **`GET /api/openapi.json`** — машиночитаемый документ (см. README субмодуля).
- **`GET /api/docs`** — Swagger UI (если включено).

Канонический источник правды — **код субмодуля**; монорепозиторий не дублирует схемы вручную в текстовых доках по каждому полю, но может хранить **снимки** JSON для диффов и обзора.

## Когда обновлять

- После **bump указателя** субмодуля, если менялся публичный HTTP API.
- По расписанию (например перед релизом платформы) или по запросу в review.

## Как снять снимок

1. Поднять сервис локально (см. `QUICKSTART.md` / `Makefile` в субмодуле) или использовать стенд с известным base URL.
2. Сохранить JSON в каталог снимков:

   ```bash
   curl -fsS "http://127.0.0.1:8000/api/openapi.json" -o docs/openapi/snapshots/auth.openapi.json
   ```

   Замените порт и имя файла на соответствующий сервис (`catalog`, `commerce`, …).

3. На Windows можно использовать скрипт из корня монорепозитория:

   ```powershell
   ./scripts/sync-openapi-snapshot.ps1 -BaseUrl "http://127.0.0.1:8000" -ServiceName "auth"
   ```

4. Закоммитить изменения отдельным коммитом, например: `docs(openapi): snapshot auth after submodule bump`.

Снимки **опциональны**: если команда не хранит JSON в git, достаточно ссылаться на живой `/api/openapi.json` в субмодуле и обновлять только обзорные документы в `docs/microservices/`.

## Где лежат снимки

Каталог `docs/openapi/snapshots/` — согласованное место для файлов `*.openapi.json`; см. [README](openapi/snapshots/README.md).

## Автоматизация

Периодический экспорт в CI (артефакты по тегам субмодулей) можно добавить отдельной задачей; текущая процедура — ручная или через скрипт выше.
