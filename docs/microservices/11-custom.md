# Custom service (`services/custom`) — детальные требования

## 1. Назначение и границы

- Владелец **конфигуратора кастомных клавиатур**: выбор совместимых компонентов, **валидация** (через Catalog rules), **черновик цены и срока**, **BOM**, статусы **согласования** с клиентом, привязка к **заказу OMS** и **производству**.
- **Не владелец**: каталожными правилами совместимости (исполняет Catalog API), оплатой (Payments), первичным созданием заказа (Commerce/OMS — координация).

## 2. Владение данными

| Сущность | Описание |
|----------|----------|
| `CustomSpecification` | id, customer_id, snapshot выбранных variant_id, атрибуты |
| `BomLine` | sku компонента, qty, позиция в дереве |
| `PricingDraft` | Разбивка цен, валюта, срок изготовления estimate |
| `ApprovalState` | draft → pending_customer → approved → locked |
| `Revision` | Версии спецификации при изменении после согласования |

## 3. API

Префикс `/api/v1`.

### Клиент

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/specs` | Создать черновик |
| GET | `/specs/{id}` | Текущее состояние |
| PATCH | `/specs/{id}` | Изменить выбор (пока не locked) |
| POST | `/specs/{id}/validate` | Вызов правил Catalog + локальные проверки |
| POST | `/specs/{id}/price` | Пересчёт цены/срока |
| POST | `/specs/{id}/submit` | Отправить клиенту на подтверждение |
| POST | `/specs/{id}/approve` | Клиент подтвердил |

### Операционка / инженерия

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/specs/{id}/lock` | Зафиксировать для производства |
| POST | `/specs/{id}/revise` | Новая ревизия |

## 4. Интеграции

- **Catalog**: `POST /compatibility/validate`, цены компонентов.
- **Commerce**: линия корзины с `custom_spec_id`.
- **OMS**: заказ содержит ссылку на зафиксированную ревизию BOM.
- **Production**: развёртывание work order из BOM.

## 5. События

| Событие | Когда |
|---------|--------|
| `custom.spec.created` | |
| `custom.spec.approved` | |
| `custom.spec.locked` | BOM неизменна для производства |

## 6. Инварианты

- После `locked` изменение только через **новую ревизию** с явным согласованием.
- BOM хранит **sku snapshot** + ссылку на каталожный variant для аудита.

## 7. Критерии приёмки

- [ ] Повторная валидация даёт тот же результат при неизменных входах.
- [ ] Цена в заказе согласована с зафиксированной ревизией.
- [ ] OpenAPI описывает все переходы статусов.
