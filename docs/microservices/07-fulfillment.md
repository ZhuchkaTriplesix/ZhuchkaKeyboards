# Fulfillment service (`services/fulfillment`) — детальные требования

## 1. Назначение и границы

- Всё **после оплаты** до **получения клиентом**: аллокация/резерв на складах (через Inventory), **волны** отгрузки, **посылки**, этикетки, интеграция с **перевозчиками**, трекинг, **RMA/возвраты** на уровне исходящей логистики.
- **Не владелец**: количественным учётом (Inventory), ячейками (WMS деталь), оплатой (Payments).

## 2. Владение данными

| Сущность | Описание |
|----------|----------|
| `Shipment` | order_id, статус, перевозчик, service level |
| `Parcel` | Вес, размеры, трек-номер, статусы сканирования у перевозчика |
| `CarrierAccount` | Учётные данные API перевозчика (в секретах) |
| `Rma` | Заявка на возврат, причина, связь с order_id |
| `Wave` / `PickBatch` | Группировка заказов на отбор (если моделируется здесь, иначе ссылка на WMS task) |

## 3. API

Префикс `/api/v1`.

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/shipments` | Создать отгрузку по `order_id` (после готовности на складе) |
| GET | `/shipments/{id}` | Статус, трек, посылки |
| POST | `/shipments/{id}/labels` | Генерация этикетки (PDF/ZPL) |
| POST | `/shipments/{id}/dispatch` | Передать в перевозчика |
| GET | `/track/{trackingNumber}` | Публичный или по токену заказа |
| POST | `/rma` | Создать RMA |
| GET | `/rma/{id}` | Статус возврата |

### Webhooks от перевозчиков

- `POST /webhooks/carriers/{name}` — статусы доставки; подпись; идемпотентность.

## 4. Интеграции

- **OMS**: источник оплаченного заказа и адреса доставки.
- **Inventory**: резерв/списание по событию или API (одна политика).
- **WMS**: задачи отбора/упаковки могут ссылаться на `shipment_id`.
- **Notifications**: события «отправлено», «задержка», «доставлено».

## 6. Нефункциональные требования

- Вызовы к API перевозчиков — **таймауты, retry с backoff**, dead-letter при исчерпании.
- Этикетки и персональные данные на этикетках — соответствие политике ПДн.

## 7. Критерии приёмки

- [ ] Один заказ не порождает две активные отгрузки без явного split.
- [ ] Webhook перевозчика не дублирует смену статуса.
- [ ] Трек доступен клиенту согласно политике (маскирование части номера опционально).
